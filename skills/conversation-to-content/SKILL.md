# When to use

Use this skill when:

* The user has a conversation or rough notes
* The goal is to extract learning and publish content
* Output includes blog or X (Twitter) post

---

# Workflow

## Step 1 — Extract Learning

Convert conversation into structured knowledge:

Output:

* Problem Definition
* Key Insights
* Mental Models
* Pitfalls
* Principles
* Takeaways

Rules:

* Do NOT summarize linearly
* Reconstruct reasoning
* Focus on reusable insights

---

## Step 2 — Critique & Improve

* Challenge weak assumptions
* Add missing trade-offs
* Sharpen unclear ideas

Output:

* Improved structured insights

---

## Step 3 — Generate Blog

Write a publish-ready technical blog.

Structure:

1. Hook (why it matters)
2. Problem framing
3. Solution approaches
4. Trade-offs
5. Recommended pattern
6. Takeaways

Style:

* Senior engineer tone
* Practical and system-oriented
* No fluff

---

## Step 4 — Generate X Post

Create ONE X post (not a thread):

Process:

1. Select the single best insight
2. Write concise, high-signal post

Style:

* Strong first line (hook)
* One idea only
* Clear and punchy
* Slightly opinionated
* Not blog-like

Output:

* Final post
* 3 alternate hooks
* 2 variants:

  * Technical
  * Provocative

---

# Global Rules

* Clarity over completeness
* High information density
* Avoid repetition
* Avoid generic statements

---

# Quality Bar

* Blog must be publish-ready
* X post must feel native
* If insights are weak → improve before writing
